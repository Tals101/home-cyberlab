# GuardDuty Findings Approach

Sample findings were skipped because the lab goal is to collect real GuardDuty findings.

Instead, this lab will use the AWS GuardDuty Tester approach.

Reason:
- GuardDuty sample findings are marked as sample data.
- GuardDuty Tester generates findings against real test resources deployed in the AWS account.
- This gives more realistic evidence for investigation, detection review, and reporting.

Safety note:
- Testing will stay inside the owned AWS lab account.
- No third-party systems will be targeted.
- Test resources will be cleaned up after the lab.
